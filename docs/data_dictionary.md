# Data dictionary

PostgreSQL schema generated by `sql/01_schema.sql` plus migration `001_add_source_system.sql`. `PK` means primary key, `FK` foreign key, and `UQ` unique.

## `customers`

One row per freight customer.

| Column | Type | Nullable | Meaning and constraints |
|---|---|---:|---|
| `customer_id` | `BIGINT IDENTITY` | No | PK |
| `customer_name` | `VARCHAR(120)` | No | Display/business name |
| `customer_type` | `VARCHAR(30)` | No | RETAILER, GROWER, DISTRIBUTOR, MANUFACTURER, or OTHER |
| `email` | `VARCHAR(255)` | Yes | UQ contact email |
| `phone` | `VARCHAR(30)` | Yes | Contact phone |
| `active` | `BOOLEAN` | No | Defaults true |
| `created_at` | `TIMESTAMPTZ` | No | Audit timestamp; defaults current time |

## `locations`

Reusable origin and destination facilities.

| Column | Type | Nullable | Meaning and constraints |
|---|---|---:|---|
| `location_id` | `BIGINT IDENTITY` | No | PK |
| `facility_name` | `VARCHAR(120)` | Yes | Warehouse/facility label |
| `address_line` | `VARCHAR(180)` | Yes | Street address |
| `city` | `VARCHAR(80)` | No | City |
| `province_state` | `VARCHAR(80)` | No | Province or state |
| `country` | `VARCHAR(80)` | No | Defaults Canada |
| `postal_code` | `VARCHAR(20)` | Yes | Postal/ZIP code |
| `created_at` | `TIMESTAMPTZ` | No | Audit timestamp |

The address, city, province/state, country, and postal-code tuple is unique.

## `drivers`

Driver master data.

| Column | Type | Nullable | Meaning and constraints |
|---|---|---:|---|
| `driver_id` | `BIGINT IDENTITY` | No | PK |
| `first_name` | `VARCHAR(60)` | No | Given name |
| `last_name` | `VARCHAR(60)` | No | Family name |
| `license_number` | `VARCHAR(50)` | No | UQ driver licence identifier |
| `hire_date` | `DATE` | No | Employment start date |
| `active` | `BOOLEAN` | No | Defaults true |
| `created_at` | `TIMESTAMPTZ` | No | Audit timestamp |

## `vehicles`

Simplified assignable fleet asset. The portfolio intentionally does not split tractors and trailers into separate assignment records.

| Column | Type | Nullable | Meaning and constraints |
|---|---|---:|---|
| `vehicle_id` | `BIGINT IDENTITY` | No | PK |
| `unit_number` | `VARCHAR(30)` | No | UQ fleet unit number |
| `vehicle_type` | `VARCHAR(40)` | No | TRACTOR, REFRIGERATED_TRAILER, DRY_VAN, or STRAIGHT_TRUCK |
| `model_year` | `SMALLINT` | No | Between 1990 and 2100 |
| `refrigerated` | `BOOLEAN` | No | Refrigeration capability |
| `active` | `BOOLEAN` | No | Availability in the fleet master |
| `created_at` | `TIMESTAMPTZ` | No | Audit timestamp |

## `shipments`

Central freight order/fact table.

| Column | Type | Nullable | Meaning and constraints |
|---|---|---:|---|
| `shipment_id` | `BIGINT IDENTITY` | No | PK |
| `reference_number` | `VARCHAR(40)` | No | UQ external-facing shipment reference |
| `customer_id` | `BIGINT` | No | FK → `customers`; delete restricted |
| `driver_id` | `BIGINT` | Yes | FK → `drivers`; set null on driver deletion |
| `vehicle_id` | `BIGINT` | Yes | FK → `vehicles`; set null on vehicle deletion |
| `origin_location_id` | `BIGINT` | No | FK → `locations`; delete restricted |
| `destination_location_id` | `BIGINT` | No | FK → `locations`; delete restricted; must differ from origin |
| `scheduled_pickup_at` | `TIMESTAMPTZ` | No | Planned pickup timestamp |
| `actual_pickup_at` | `TIMESTAMPTZ` | Yes | Recorded pickup timestamp |
| `scheduled_delivery_at` | `TIMESTAMPTZ` | No | Planned delivery; must be after scheduled pickup |
| `actual_delivery_at` | `TIMESTAMPTZ` | Yes | Recorded delivery; cannot precede actual pickup |
| `cargo_type` | `VARCHAR(80)` | No | Synthetic freight class |
| `weight_kg` | `NUMERIC(10,2)` | No | Positive gross cargo weight |
| `agreed_revenue` | `NUMERIC(12,2)` | No | Non-negative contracted revenue |
| `estimated_cost` | `NUMERIC(12,2)` | No | Non-negative estimated operating cost |
| `required_temperature_c` | `NUMERIC(5,2)` | Yes | Cold-chain target from -40°C to 25°C; null for dry goods |
| `created_at` | `TIMESTAMPTZ` | No | Audit timestamp |
| `source_system` | `VARCHAR(30)` | No | Added by migration 001; SYNTHETIC_GENERATOR, TMS, EDI, or MANUAL |

There is deliberately no duplicated current-status column. Current status is derived from the latest row in `shipment_status_history` and exposed by `vw_shipment_current_status`.

## `shipment_status_history`

Append-only lifecycle events.

| Column | Type | Nullable | Meaning and constraints |
|---|---|---:|---|
| `status_id` | `BIGINT IDENTITY` | No | PK and deterministic timestamp tie-breaker |
| `shipment_id` | `BIGINT` | No | FK → `shipments`; cascades on shipment deletion |
| `status` | `VARCHAR(30)` | No | CREATED, ASSIGNED, PICKED_UP, IN_TRANSIT, DELAYED, DELIVERED, or CANCELLED |
| `status_time` | `TIMESTAMPTZ` | No | Business event timestamp |
| `notes` | `TEXT` | Yes | Event detail |
| `created_at` | `TIMESTAMPTZ` | No | Row audit timestamp |

The shipment, status, and status-time tuple is unique. `idx_status_latest` supports latest-row retrieval by `(shipment_id, status_time DESC, status_id DESC)`.

## `temperature_readings`

Cold-chain telemetry associated with picked-up temperature-controlled shipments.

| Column | Type | Nullable | Meaning and constraints |
|---|---|---:|---|
| `reading_id` | `BIGINT IDENTITY` | No | PK |
| `shipment_id` | `BIGINT` | No | FK → `shipments`; cascades on shipment deletion |
| `recorded_at` | `TIMESTAMPTZ` | No | Sensor observation time |
| `temperature_c` | `NUMERIC(5,2)` | No | Observed temperature between -80°C and 60°C |
| `sensor_id` | `VARCHAR(50)` | No | Sensor identifier |
| `created_at` | `TIMESTAMPTZ` | No | Row audit timestamp |

The shipment, sensor, and recorded-time tuple is unique and prevents duplicate telemetry events.

## `invoices`

At most one invoice per delivered shipment.

| Column | Type | Nullable | Meaning and constraints |
|---|---|---:|---|
| `invoice_id` | `BIGINT IDENTITY` | No | PK |
| `invoice_number` | `VARCHAR(40)` | No | UQ invoice reference |
| `shipment_id` | `BIGINT` | No | UQ FK → `shipments`; delete restricted |
| `invoice_date` | `DATE` | No | Issue date |
| `due_date` | `DATE` | No | Must be on/after invoice date |
| `amount` | `NUMERIC(12,2)` | No | Non-negative billed amount |
| `payment_status` | `VARCHAR(20)` | No | PENDING, PAID, OVERDUE, or VOID |
| `paid_at` | `TIMESTAMPTZ` | Yes | Required only when status is PAID; forbidden otherwise |
| `created_at` | `TIMESTAMPTZ` | No | Audit timestamp |

`idx_invoices_outstanding_due` is a partial index for PENDING/OVERDUE collection work.

## Reporting views

| View | Grain | Purpose |
|---|---|---|
| `vw_shipment_current_status` | One row per shipment | Latest lifecycle status and event detail |
| `vw_customer_delivery_performance` | One row per customer | Shipment volume, reliability, and delivered revenue |
| `vw_invoice_aging` | One row per invoice | Reproducible aging bucket as of 2026-07-16 |
